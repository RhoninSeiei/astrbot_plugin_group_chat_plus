## 已确认的关键决策

1. 采用“从现有插件直接分叉并做减法”的方案。
2. 新插件固定只服务 `matoi_guardian_ep5_cn`。
3. 新插件强制群白名单，旧插件保持现状，由配置层人工避免群号重叠。
4. 决策、回复、主动发言统一强制绑定 `matoi_guardian_ep5_cn`。
5. 保留主动发言，但改成更贴近角色扮演的触发条件与提示词。
6. 数据目录、缓存、记忆命名空间全部独立。
7. 采用混合命名策略：核心配置键尽量沿用，插件名、指令前缀、日志标识换新。
8. 移除 `web/` 与私聊模块。
9. 增加长期实施记录文档，用于维护和上下文压缩参考。

## 已知问题与关注点

1. 当前 `chat_plus` 主线体量较大，首轮裁剪需要控制范围，避免误删主线依赖。
2. 角色扮演提示词改写后，主动发言和读空气判断的行为会发生明显变化，需要专项验证。
3. 若 LivingMemory 的人格隔离方式与预期不完全一致，需要补一层插件侧命名空间约束。
4. 旧插件继续服务其他群，因此任何实现都不能要求旧插件同步修改。

## 阶段摘要

当前项目已经完成设计确认。整体方向是从 `chat_plus` 主线直接分叉，保留已验证过的群聊主干能力，删除 Web 和私聊等非核心部分，并把插件改造成一个只服务 `matoi_guardian_ep5_cn` 的角色扮演专用群聊插件。新插件使用独立仓库、独立插件名、独立数据目录、独立记忆命名空间，通过强制群白名单与旧插件并存。

后续实施时，优先顺序是先完成身份隔离与非核心模块删除，再接入固定人格、强制白名单和角色扮演提示词，最后做并存与稳定性验证。
